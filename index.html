JSON Form Generator and Editor

A lightweight JavaScript library for generating forms from JSON/Object.
Installation Usage Methods Events and Filters Examples Bootstrap Template Custom Labels Exclude Path Field Attributes Form Validation Radio and Select Datatable

Installation

To get started, you can choose one of the options below to install the JF.

Install via NPM 

npm install @rmanaf/json-form

From CDN

Copy the code and paste it into the <head> tag of your HTML document.

<script src="//cdn.jsdelivr.net/npm/@rmanaf/json-form@latest/dist/json-form.min.js" 
        integrity="sha384-92PVCBjuXieCHzzoJ+olL3PawBp25IQzPoGbNHVAMhjHSm9Krdu8qjP0sZr/0qWP" 
        crossorigin="anonymous" ></script>

Download the repository

You can also download the repository and upload the dist folder to your server.

To clone the repository run the following command:

gh repo clone rmanaf/json-form

Usage

You can use the JF constructor to create a new instance of the editor. 

 new JsonForm(data: string | object, options?: object , target?: string);
Data

The data can be either a JSON string or an object.

// Using JSON string
const editor_1 = new JsonForm("{ key : value }");

// Or using object
var data = { key : "value" }
const editor_2 = new JsonForm(data);
Options

The following options are supported:

  • body: HTMLElement , default: document.body

    The element to which the form content will be appended.

  • model: string , default: " "

    The model specifies which subset of the input data to use for form generation.

  • exclude: string[ ] , default : [ ]

    Paths that should not be in the form. (e.g. data.model.property or /.+/)

  • labels: object , default : { }

    List of labels associated with each field. If you are using templates, you can use the {{label}} to display the label.

  • sections: object , default : { }

    You can use the sections property to group fields. The section list contains the section name as the key and an object as the value that supports the following properties:

    • children: string[ ] , default : [ ]

      A list containing paths that will be grouped in this section. (e.g. data.model.property or /.+/)

    • template: string, default : "section"

      The node name of this section. (e.g. tr) or HTML string (e.g. <tr></tr>).

    • repeat: boolean , default : false

      If true, the section will be repeated for each child.

    • merge: boolean, default : false

      For array of objects, all the properties of each object will be placed in this section.

  • types: object , default : { }

    The type that will be used for input. By default, the input type is associated with the property type.

    Property Type Result
    String <input type="text"/>
    Number <input type="number"/>
    Boolean <input type="checkbox"/>

    You can use the {{inputType}} to put the input type in the template.

  • attributes: object , default : { }

    The list of attributes for each input. For example, you can use the placeholder attribute to set the placeholder for an input. 

    const options = {
    attributes: {
        "foo.bar": {
            placeholder: "Enter a value..."
        }
    }
}

  • templates: object , default : { }

    List of template IDs to use for the fields. Here is an example of defining a template:
    Options: 

    const options = {
    templates: {
        "data.path.to.property": "template-id"
    }
}
    Template: 
    <template id="template-id">
    <input id={{id}} type="{{inputType}}" placeholder="Enter a value...">
</template>

  • meta: object , default : { }

    To define more variables for use in templates, you can use the meta attribute. The meta value can be a function. 

    const options = {
    meta : { 
        "path.to.property" : { 
            "metaName" : "metaValue" 
        },
        "foo.bar" : {
            "path" : (args) => {
                // args.id is the element ID.
                // args.inputType is the input type. (text, number, password, checkbox and etc.)
                // args.type is the property type.
                // args.label is the input label.
                // args.value is the current value of the input.
                // args.path is the propery path.

                return args.path;
            }
        }
    }
}

  • events: object , default : { "*" : "keyup keypress blur change", "*-number" : "keyup keypress blur change mouseup" }

    The input events. You can define custom events for specific inputs.

Target optional

JF will automatically print the form values in the target element. You can use it for debugging or sending JSON through a form.
